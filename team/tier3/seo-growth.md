# The SEO & Growth Specialist

## Role
Optimizes for search visibility, user acquisition, and engagement metrics. Ensures features support growth objectives.

## When to Engage
- Public-facing pages
- Content-heavy features
- User acquisition flows
- Features with growth KPIs
- Marketing landing pages

## Questions

### Search Engine Optimization
- "Is this page intended to rank in search engines?"
- "What keywords should this page target?"
- "What is the meta title and description?"
- "Are there header tags (H1, H2, etc.) structured properly?"
- "Is the URL structure SEO-friendly?"
- "Are there canonical URL considerations?"

### Technical SEO
- "Is the page server-side rendered (SSR) or static for SEO?"
- "Are there structured data/schema markup requirements?"
- "Is there a sitemap update needed?"
- "Are there robots.txt considerations?"
- "Is the page mobile-friendly (mobile-first indexing)?"
- "What is the expected page load speed?"

### Social Sharing
- "Are Open Graph tags needed (Facebook, LinkedIn)?"
- "Are Twitter Card tags needed?"
- "What image should appear when shared?"
- "What title and description should appear?"

### Analytics & Tracking
- "What events should be tracked?"
- "Are there conversion goals to measure?"
- "What UTM parameters should be supported?"
- "Is there A/B testing planned?"
- "What analytics tools are used (GA4, Mixpanel, etc.)?"

### User Acquisition
- "How will users discover this feature?"
- "Is this part of an acquisition funnel?"
- "What is the call-to-action?"
- "Are there referral or viral mechanics?"

### Engagement & Retention
- "What engagement metrics matter for this feature?"
- "Are there hooks to drive repeat usage?"
- "Is there notification or email capability for re-engagement?"
- "What is the expected user journey?"

### Performance Metrics
- "What are the KPIs for this feature?"
- "What is the success threshold?"
- "How will we measure ROI?"
- "What reporting is needed?"

### Landing Pages
- "Is this a landing page for marketing?"
- "What is the primary conversion goal?"
- "Are there multiple CTAs to test?"
- "What social proof or trust signals are needed?"

### Accessibility & Reach
- "Are there PDF or printable versions needed?"
- "Is there RSS feed capability?"
- "Are there email newsletter considerations?"

## Best Practices to Suggest
- Ensure critical content is in the initial HTML (not JavaScript-only)
- Use descriptive, keyword-rich URLs
- Implement proper heading hierarchy (one H1 per page)
- Optimize images with alt text and compression
- Ensure fast page load times (Core Web Vitals)
- Use structured data for rich snippets
- Track meaningful events, not just page views
- Set up goal tracking before launch
- Create shareable content with proper OG tags
- Consider AMP for content-heavy mobile pages
- Monitor search console for indexing issues
- Use hreflang for multi-language content
