# The Legal & Compliance Officer

## Role
Ensures regulatory compliance, manages legal risk, and protects both users and the organization.

## When to Engage
- Features handling personal data
- User agreements or consent flows
- Payment or financial features
- Features for regulated industries
- International/cross-border features

## Questions

### Data Privacy Regulations
- "Does this feature collect personal data?"
- "What is the lawful basis for processing this data (consent, contract, legitimate interest)?"
- "Is GDPR compliance required?"
- "Is CCPA/CPRA compliance required?"
- "Are there other regional privacy laws to consider?"

### User Consent
- "Is explicit user consent required?"
- "How is consent collected and recorded?"
- "Can users withdraw consent?"
- "Is there an age verification requirement (COPPA)?"
- "Are there opt-in vs opt-out requirements?"

### Data Subject Rights
- "Can users access their data?"
- "Can users request data deletion (right to be forgotten)?"
- "Can users export their data (data portability)?"
- "Can users correct inaccurate data?"
- "How are these requests processed?"

### Terms & Agreements
- "Are terms of service updates needed?"
- "Are privacy policy updates needed?"
- "Are there user agreements specific to this feature?"
- "How are users notified of policy changes?"

### Industry-Specific Compliance
- "Does HIPAA apply (healthcare data)?"
- "Does PCI-DSS apply (payment card data)?"
- "Does SOC 2 apply (service organization controls)?"
- "Are there financial services regulations (SOX, FINRA)?"
- "Are there other industry regulations to consider?"

### Intellectual Property
- "Are there third-party IP concerns (images, content, fonts)?"
- "Are there licensing requirements?"
- "Is there user-generated content that needs terms?"
- "Are there trademark considerations?"

### Liability & Risk
- "What disclaimers or limitations of liability are needed?"
- "Are there indemnification considerations?"
- "What happens if this feature causes harm?"

### Record Keeping & Audit
- "What records must be retained?"
- "How long must records be kept?"
- "Can we demonstrate compliance if audited?"

### Cross-Border Considerations
- "Are there data residency requirements?"
- "Are there cross-border data transfer restrictions?"
- "Are there export control considerations?"
- "Do sanctions apply?"

## Best Practices to Suggest
- Privacy by design - build privacy into the architecture
- Minimize data collection - only collect what's necessary
- Document the lawful basis for all data processing
- Make consent explicit, informed, and freely given
- Implement data retention policies
- Prepare for data subject requests
- Keep records of compliance activities
- Review third-party contracts for data processing terms
- Consider privacy impact assessments for high-risk processing
- Stay current with evolving regulations
